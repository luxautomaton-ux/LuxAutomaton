Getting Started with Lux AI Studio

Developed by Lux Automaton. Welcome to the Lux AI Ecosystem. This guide will help you set up and start using your local AI operating stack.

What You'll Need

  • ✓ macOS 13+, Windows 11, or Linux
  • ✓ 16GB RAM minimum (64GB recommended)
  • ✓ Apple Silicon, NVIDIA GPU, or AMD GPU
  • ✓ 20GB free disk space

Installation

Download and install Lux AI Studio on your machine.

# Clone the repository

git clone https://github.com/luxautomaton/lux-ai-studio.git

cd lux-ai-studio

# Install dependencies

npm install

# Start the application

npm run dev

The app will be available at http://localhost:5175

Configuration

Configure your environment to get the most out of Lux AI Studio.

# Create .env file in your project root

DEFAULT_MODEL=gemma-4

OLLAMA_HOST=http://localhost:11434

CLAUDE_CODE_USE_OPENAI=1

Environment Variables

DEFAULT_MODEL

The default model used for inference. Options: gemma-4, qwen2.5, llama3

OLLAMA_HOST

The host URL for your Ollama instance

CLAUDE_CODE_USE_OPENAI

Set to 1 to use local Opencode instead of cloud API

Using Agents

Each agent has specific capabilities. Here's how to use them effectively.

Manus — Autonomous Research

Best for research tasks, web crawling, and complex automation.

lux agent start --agent=manus --task="research topic"

Claude Code — Coding Assistant

Best for writing, reviewing, and debugging code.

lux agent start --agent=claude --mode=code

CoWork — Collaboration

Best for team workflows and multi-agent coordination.

lux cowork start --workflow=approval

Hermes — Chat Interface

Best for interactive conversations with agents.

lux hermes start --port=8787

Privacy & Security

Your data never leaves your machine. Here's how we ensure privacy.

Local-First Architecture

  • ✓ All prompts stored locally in SQLite
  • ✓ Agent sessions saved as JSON
  • ✓ No cloud authentication required
  • ✓ Air-gapped system compatible

Troubleshooting

Common issues and how to resolve them.

Model not loading

Make sure Ollama is running: ollama serve

Port already in use

Check for conflicting processes: lsof -i :5175

Memory issues

Switch to a smaller model or increase system RAM
