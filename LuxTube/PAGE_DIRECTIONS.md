# Lux Tube Page Directions

This guide mirrors the in-app `Page Guide` tab.

## 1) Dashboard

1. Confirm runtime health and local engine readiness.
2. Use `Refresh Data` after changing stack settings.
3. Resolve any missing runtime dependency before production work.

## 2) Viral Scanner

1. Paste 3-15 competitor channels.
2. Run scanner to extract winning cadence, hooks, and title patterns.
3. Use scanner evidence as source input for planning tabs.

## 3) Shorts Planner

1. Generate shorts plan from current scanner evidence.
2. Publish multiple hook variants for the same concept.
3. Keep only top retention performers.

## 4) Long-form Planner

1. Generate one authority-focused long-form blueprint.
2. Structure sections around retention resets and one core outcome.
3. Convert long-form highlights into shorts after publish.

## 5) AI Models

1. Review recommended local model.
2. Apply selected model to cross-stack config files.
3. Use auto-set for one-click stack tuning.

## 6) HeyGen Studio

1. Connect `HEYGEN_API_KEY`.
2. Validate CLI, API key, and MCP status.
3. Keep Hyperframes as the default local engine.

## 7) Skills + Profile

1. Set profile fields: niche, voice, goal, CTA.
2. Install safe skill pack before adding custom skills.
3. Revisit profile weekly to match channel evolution.

## 8) Video Generator

1. Write a production-ready script per job.
2. Select engine: Hyperframes (default), HeyGen, or local pipeline.
3. Track returned job metadata in your production workflow.

## 9) Page Guide

1. Use as canonical checklist for all Lux Tube pages.
2. Follow startup and execution sequence exactly.
3. Repeat scanner -> strategy -> production every week.

## Recommended Weekly Loop

1. Scanner evidence first.
2. Shorts and long-form planning second.
3. Video production third.
4. Metrics review and iteration last.
